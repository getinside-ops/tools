Design — Section 2: Migration scope

Two decisions that affect the implementation plan

Decision 1 — "About" text for 30 tools

Who writes the content?

A

I write all 30 about texts

Claude generates a short paragraph (2–3 sentences) for each tool based on its name, description, and i18n translations already in the codebase. You review and adjust.

✓ Zero work for you  ✓ Done in one pass

B

Stub placeholder, you fill later

The #about slot is wired up but contains a TODO placeholder in each view. You write the real text when ready.

Faster implementation, more work for you later.

Decision 2 — Per-view inline style cleanup

Several views use inline style="" attributes instead of scoped CSS

A

Fix inline styles in the same pass

While updating each view for the new ToolPageLayout props, also move inline styles to scoped CSS classes. Affected views: MatteGeneratorView, ImageResizerView, ContrastCheckerView, PxToRemView (+ migrate PxToRemView to ToolPageLayout).

✓ One clean pass  ✓ Better maintainability

B

Only add category + about, leave styles as-is

Minimal change: only add the new category prop and #about slot to each view. Don't touch existing layout styles.

Faster, but misses the consistency goal on the worst offenders.
